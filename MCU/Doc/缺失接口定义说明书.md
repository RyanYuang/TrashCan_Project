## 接口缺失概览

根据对现有 `Driver` 和 `Device` 层的分析，气体巡检小车功能实现需要以下**5类关键接口**：

| 序号 | 接口类别 | 优先级 | 状态 | 说明 |
|------|---------|--------|------|------|
| 1 | 红外循迹传感器 | **必需** | ❌ 缺失 | 循迹功能的核心传感器 |
| 2 | 超声波距离读取 | **必需** | ⚠️ 不完整 | 仅有触发函数，缺少距离读取 |
| 3 | 气体传感器MQ-2 | **必需** | ⚠️ 不完整 | 缺少ADC数据获取接口 |
| 4 | 温湿度传感器AHT30 | 可选 | ❌ 缺失 | 环境数据采集（非核心功能） |
| 5 | OLED引脚配置 | **必需** | ⚠️ 不完整 | 缺少引脚和地址配置信息 |

## 1. 红外循迹传感器接口

### 📌 功能需求

实现红外循迹功能，需要读取**3路红外传感器**的状态（左、中、右），用于判断小车相对黑线的位置。

### 🔧 所需接口

#### 1.1 单个红外传感器读取

```c
/**
 * @brief 读取单个红外循迹传感器状态
 * @param ir_port GPIO端口（如 GPIOA, GPIOB）
 * @param ir_pin GPIO引脚（如 GPIO_PIN_0）
 * @retval bool true: 检测到黑线, false: 未检测到（白色表面）
 * @note 红外传感器通常为数字量输出：
 *       - 低电平(0): 检测到黑线（反射弱）
 *       - 高电平(1): 检测到白色（反射强）
 *       根据实际传感器极性可能需要反向
 */
bool IR_Sensor_Read(GPIO_TypeDef* ir_port, uint16_t ir_pin);
```

#### 1.2 三路循迹传感器读取（推荐）

```c
/**
 * @brief 读取三路红外循迹传感器状态
 * @param left_state 指向左侧传感器状态的指针（输出）
 * @param center_state 指向中间传感器状态的指针（输出）
 * @param right_state 指向右侧传感器状态的指针（输出）
 * @retval int 0: 成功, -1: 失败
 * @note 状态值：true=检测到黑线, false=未检测到
 */
int IR_Tracking_Read_All(bool* left_state, bool* center_state, bool* right_state);
```

### 📍 硬件连接要求

请提供以下信息：

1. **左侧红外传感器** GPIO引脚：`GPIOx, GPIO_PIN_x`
2. **中间红外传感器** GPIO引脚：`GPIOx, GPIO_PIN_x`
3. **右侧红外传感器** GPIO引脚：`GPIOx, GPIO_PIN_x`
4. **传感器输出极性**：低电平有效 / 高电平有效

### 💡 应用层调用示例

```c
// 在 EnvCar_Sensor_Update() 中调用
bool left, center, right;
IR_Tracking_Read_All(&left, &center, &right);

g_Sensor_Data.ir_left = left;
g_Sensor_Data.ir_center = center;
g_Sensor_Data.ir_right = right;
```

## 2. 超声波距离读取接口

### 📌 功能需求

现有 `ultrasonic.h` 中仅提供了 `ultrasonic_task1()` 和 `ultrasonic_task2()` 触发函数，**缺少读取距离数据的接口**。

避障功能需要实时获取两个超声波传感器测量的距离值（单位：厘米）。

### 🔧 所需接口

#### 2.1 读取超声波1距离

```c
/**
 * @brief 获取超声波传感器1测量的距离
 * @retval uint16_t 距离值（单位：厘米）
 * @note 返回值范围：2cm ~ 400cm（根据HC-SR04规格）
 *       如果测量失败或超出范围，返回 0 或 0xFFFF
 */
uint16_t Ultrasonic_Get_Distance_1(void);
```

#### 2.2 读取超声波2距离

```c
/**
 * @brief 获取超声波传感器2测量的距离
 * @retval uint16_t 距离值（单位：厘米）
 * @note 返回值范围：2cm ~ 400cm
 *       如果测量失败或超出范围，返回 0 或 0xFFFF
 */
uint16_t Ultrasonic_Get_Distance_2(void);
```

### 📍 实现原理说明

超声波测距通常基于**定时器输入捕获**实现：

1. `ultrasonic_taskX()` 发送Trig脉冲触发测距
2. Echo引脚产生高电平脉冲，脉宽与距离成正比
3. 定时器捕获Echo脉冲的上升沿和下降沿时间
4. 计算距离：`距离(cm) = (高电平时间(μs) × 声速(340m/s)) / 2 / 10000`

**请在驱动层实现上述计算逻辑，并提供读取接口。**

### 💡 应用层调用示例

```c
// 在 EnvCar_Sensor_Update() 中调用
g_Sensor_Data.ultrasonic1_distance_cm = Ultrasonic_Get_Distance_1();
g_Sensor_Data.ultrasonic2_distance_cm = Ultrasonic_Get_Distance_2();
```

## 3. 气体传感器MQ-2接口

### 📌 功能需求

现有 `MQ-2.h` 中仅提供了 `MQ2ConvertPPM()` 转换函数，**缺少ADC原始数据获取接口**。

气体监测功能需要：
1. 获取MQ-2传感器的ADC采样值
2. 将ADC值转换为气体浓度（ppm）

### 🔧 所需接口

#### 3.1 读取MQ-2 ADC原始值

```c
/**
 * @brief 读取MQ-2气体传感器ADC原始值
 * @retval uint32_t ADC采样值（12位ADC: 0~4095）
 * @note 需要启动ADC转换并等待转换完成
 *       建议使用DMA方式提高效率
 */
uint32_t MQ2_Get_ADC_Value(void);
```

#### 3.2 读取MQ-2气体浓度（推荐）

```c
/**
 * @brief 读取MQ-2气体浓度（ppm）
 * @retval float 气体浓度值（单位：ppm）
 * @note 内部调用 MQ2_Get_ADC_Value() 和 MQ2ConvertPPM()
 *       可根据实际传感器校准系数调整转换公式
 */
float MQ2_Get_Concentration_PPM(void);
```

### 📍 硬件连接要求

请提供以下信息：

1. **MQ-2 ADC通道**：`ADCx, ADC_CHANNEL_x`
2. **ADC采样分辨率**：12位 / 10位 / 8位
3. **参考电压**：3.3V / 5V
4. **传感器校准参数**（如有）

### 💡 应用层调用示例

```c
// 在 EnvCar_Sensor_Update() 中调用
g_Sensor_Data.gas_adc_value = MQ2_Get_ADC_Value();
g_Sensor_Data.gas_concentration_ppm = MQ2ConvertPPM(g_Sensor_Data.gas_adc_value);

// 或直接调用
g_Sensor_Data.gas_concentration_ppm = MQ2_Get_Concentration_PPM();
```

## 4. 温湿度传感器AHT30接口

### 📌 功能需求

AHT30温湿度传感器用于采集环境数据，虽然**不是核心功能**，但对完整的数据监控有帮助。

### 🔧 所需接口

#### 4.1 读取温度

```c
/**
 * @brief 读取AHT30温度数据
 * @retval float 温度值（单位：摄氏度 ℃）
 * @note 测量范围：-40℃ ~ 85℃
 *       精度：±0.3℃
 */
float AHT30_Get_Temperature(void);
```

#### 4.2 读取湿度

```c
/**
 * @brief 读取AHT30湿度数据
 * @retval float 相对湿度值（单位：%RH）
 * @note 测量范围：0% ~ 100% RH
 *       精度：±2% RH
 */
float AHT30_Get_Humidity(void);
```

#### 4.3 一次性读取温湿度（推荐）

```c
/**
 * @brief 一次性读取AHT30温湿度数据
 * @param temperature 指向温度变量的指针（输出）
 * @param humidity 指向湿度变量的指针（输出）
 * @retval int 0: 成功, -1: 失败
 * @note AHT30一次测量同时返回温湿度，建议一次读取提高效率
 */
int AHT30_Read_Temperature_Humidity(float* temperature, float* humidity);
```

### 📍 硬件连接要求

AHT30为**IIC接口**，请确认：

1. **IIC地址**：0x38（默认） / 0x39
2. **IIC引脚**：SDA和SCL已在 `aht30.h` 中配置

### 💡 应用层调用示例

```c
// 在 EnvCar_Sensor_Update() 中调用
AHT30_Read_Temperature_Humidity(&g_Sensor_Data.temperature, &g_Sensor_Data.humidity);
```

1. **IIC引脚配置**（SDA和SCL）
2. **OLED设备地址**

### 🔧 所需信息

请提供以下硬件配置信息：

#### 5.1 OLED IIC引脚

```c
// 示例：

Pin_Struct oled_scl = {GPIOB, GPIO_PIN_6};  // 请替换为实际引脚
```

#### 5.2 OLED IIC地址

```c
// OLED常用地址：
// 0x78 (7位地址: 0x3C)
// 0x7A (7位地址: 0x3D)
uint8_t oled_address = 0x78;  // 请确认实际地址
```

### 💡 应用层初始化示例


## 📊 接口优先级总结

### 🔴 高优先级（必需）

1. **红外循迹传感器接口** - 无此接口无法实现循迹功能
2. **超声波距离读取接口** - 无此接口无法实现避障功能
3. **MQ-2 ADC读取接口** - 无此接口无法实现气体监测功能
4. **OLED引脚配置信息** - 无此信息无法初始化OLED显示

### 🟡 中优先级（建议）

5. **AHT30温湿度接口** - 增强数据完整性，非核心功能

```

└── AHT30/               # 新增：AHT30驱动接口
    ├── aht30.h
    └── aht30.c
```

### 驱动层与Device层职责分工

- **Driver层**：提供底层硬件抽象（GPIO、ADC、IIC、Timer等）
- **Device层**：基于Driver层实现具体传感器的数据获取和控制接口
- **App层**：调用Device层接口实现应用逻辑，不直接操作硬件
