# TrashCan Project 系统框架说明

## 1. 总体分层

本项目按“硬件驱动 -> 设备适配 -> 业务应用 -> 中断调度”四层组织：

1. 驱动层（Driver / HAL）
- HAL 外设初始化与底层驱动：`Core/Src/*.c`、`Drivers/STM32F1xx_HAL_Driver/*`
- 软件驱动封装：`Driver/Driver_IIC`、`Driver/Driver_Timer`

2. 设备层（Device）
- 传感器：`Device/AHT30`、`Device/GY302`、`Device/MQ-2`、`Device/ultrasonic.c`
- 执行器：`Device/TB6612`、`Device/Beep`、`Device/RGB`、`Device/OLED`
- 统一入口：`Device/Devices.h`

3. 应用层（App + Core 业务模块）
- 主业务框架：`App/User_Main.c`
- 巡逻状态机：`App/patrol.c`
- 协议与配置：`Core/Src/uart_protocol.c`、`Core/Src/protocol_parser.c`、`Core/Src/threshold_config.c`

4. 中断层（ISR）
- 统一中断入口：`Core/Src/stm32f1xx_it.c`
- UART 接收回调、TIM6 上报节拍、TIM8 超声波捕获都在此汇总

---

## 2. 启动流程（Boot Sequence）

### 2.1 系统启动
1. `main()` 完成 HAL 与外设初始化（GPIO/ADC/UART/TIM/DMA）
2. 进入 `User_Main()`

### 2.2 应用初始化（`User_SystemFramework_Init`）
1. 初始化阈值系统：`ThresholdConfig_Init`
2. 初始化协议解析：`Protocol_Parser_Init`
3. 注册命令回调：`Protocol_Parser_RegisterCommandCallback`
4. 启动串口接收：`UART_Protocol_Init(&huart2)`
5. 初始化设备：OLED/GY302/AHT30/TB6612/ultrasonic/patrol
6. 启动节拍定时：`HAL_TIM_Base_Start_IT(&htim6)`

---

## 3. 运行框架（Main Loop Framework）

主循环为轮询调度 + 中断事件驱动混合模型：

```c
while (1) {
    User_Task_ReadSensors();
    User_Task_ExecutePatrol();
    User_Task_ReportAndDisplay();
    User_Task_HandleAlert();
    HAL_Delay(10);
}
```

### 3.1 周期任务职责

1. `User_Task_ReadSensors`
- 读取 GY302 光照
- DMA 采样 MQ-2 并转换 PPM
- 读取 AHT30 温湿度

2. `User_Task_ExecutePatrol`
- 触发超声波测距（`ultrasonic_task1`）
- 执行巡逻状态机（`Patrol_Task`）

3. `User_Task_ReportAndDisplay`
- 由 `g_DataReportFlag` 触发（TIM6 中断置位）
- 串口上报环境数据
- OLED 刷新展示

4. `User_Task_HandleAlert`
- 组装 `SensorData_t`
- 阈值判定 `ThresholdConfig_CheckAlert`
- 控制蜂鸣器与 RGB

---

## 4. 中断与事件框架

### 4.1 UART 中断事件链
1. `HAL_UART_RxCpltCallback`（`stm32f1xx_it.c`）
2. `UART_Protocol_RxCallback`：组帧（按 `\n` 结束）
3. 完整帧就绪 -> `UART_Protocol_GetFrame`
4. `Protocol_Parse` 分发：
- `@<cmd>` 控制命令
- `#T...` 阈值配置

### 4.2 TIM6 周期事件
- `TIM6_IRQHandler` 周期置位 `g_DataReportFlag`
- 主循环检测标志后再做上报/显示（中断中不做重计算）

### 4.3 TIM8 捕获事件
- `HAL_TIM_IC_CaptureCallback` 更新超声波距离
- 巡逻状态机读取距离后决策避障

---

## 5. 行为状态机框架

### 5.1 巡逻状态机（`App/patrol.c`）
1. `PATROL_STATE_IDLE`
2. `PATROL_STATE_FORWARD`
3. `PATROL_STATE_TURN`
4. `PATROL_STATE_AVOID_STOP`

状态转换由“动作超时 + 障碍物检测”触发，使用 `HAL_GetTick()` 实现非阻塞时间基准。

### 5.2 告警状态机（逻辑型）
1. `ALERT_NONE` -> 绿灯
2. `ALERT_WARNING` -> 红灯 + 蜂鸣
3. `ALERT_DANGER` -> 持续蜂鸣 + 红灯

---

## 6. 数据流框架

1. 采集：AHT30 / MQ-2 / GY302 / ultrasonic
2. 融合：主循环构造 `SensorData_t`
3. 决策：
- 阈值告警决策（`ThresholdConfig_CheckAlert`）
- 巡逻避障决策（`Patrol_Task`）
4. 输出：
- 执行器控制（TB6612 / Beep / RGB）
- OLED 展示
- 串口上报 `@temp,hum,ppm,lux,obstacle`

---

## 7. 对外扩展入口（推荐）

后续新增功能时，按以下入口扩展，避免改乱：

1. 新增传感器
- 设备驱动放在 `Device/<module>/`
- 在 `User_Task_ReadSensors` 增加采样
- 在 `User_BuildSensorData` 或上报格式中加入字段

2. 新增行为模式（如自动回充）
- 新建 `App/<mode>.c/.h` 状态机
- 在 `User_Task_ExecutePatrol` 同级挂载 `User_Task_ExecuteXXX`

3. 新增串口命令
- 在 `ControlCommand_t` 增加枚举
- 在 `Parse_ControlCommand` 扩展解析范围
- 在 `OnCommandReceived` 增加执行动作

4. 新增告警策略
- 优先改 `threshold_config.c`
- `User_Task_HandleAlert` 仅保留执行映射，不塞复杂算法

---

## 8. 当前框架的边界约束

1. 主循环里存在 `HAL_Delay(500)`（告警 WARNING 分支），会降低实时性
2. `User_Task_ReadSensors` 采用轮询读法，采样周期由主循环速度决定
3. 数据上报节拍由 TIM6 置位标志控制

如果后续要做高实时版本，建议把告警蜂鸣改为“非阻塞节拍状态机”。

---

## 9. 推荐的文件责任边界

1. `App/User_Main.c`
- 只做系统编排（init + task scheduler + task glue）

2. `App/patrol.c`
- 只做巡逻与避障状态机

3. `Core/Src/protocol_parser.c`
- 只做协议语义解析与回调分发

4. `Core/Src/threshold_config.c`
- 只做阈值策略与告警级别判断

5. `Device/*`
- 只做硬件能力，不做业务策略
